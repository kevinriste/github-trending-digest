GhostLock (CVE-2026-43499) is a critical use-after-free vulnerability that resided in the Linux kernel's real-time mutex (rtmutex) subsystem for 15 years, affecting all major distributions since 2011. The flaw occurs during a specific rollback path in the `FUTEX_CMP_REQUEUE_PI` syscall, which incorrectly clears a task's blocking pointer and creates a dangling reference to kernel stack memory. By leveraging techniques like stack-frame reclamation and CPU Entry Area (CEA) spraying, an unprivileged local attacker can transform this dangling pointer into an arbitrary write primitive to hijack kernel control flow. This vulnerability is significant because it requires no special privileges or non-standard configurations to exploit, eventually granting an attacker root-level access and container escape capabilities.
