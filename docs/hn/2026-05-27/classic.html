Analysis

CVE-2026-48710 is a critical vulnerability in Starlette versions prior to 1.0.1 that allows attackers to bypass path-based authentication by manipulating the `Host` header to forge the `request.url.path`. Because Starlette insecurely constructs URLs by concatenating the raw `Host` header with the request path, custom middleware relying on path-based security checks can be tricked into granting unauthorized access. The flaw affects numerous high-profile Python frameworks and AI infrastructure tools, including FastAPI, vLLM, and LiteLLM, particularly when they are deployed without an RFC-compliant reverse proxy. Remediation requires updating to Starlette 1.0.1 or modifying middleware to use the `scope["path"]` attribute, which is not susceptible to header injection. This discovery highlights the inherent risks of layering independent specifications that, while correct in isolation, create dangerous vulnerabilities when combined.

Comment Analysis
