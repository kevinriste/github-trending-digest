Core // Developer Tooling

Optimize AI coding contexts with AST-based graph mapping

github.com  ·  25,027 stars  ·  1,925 stars today

Code-review-graph uses Tree-sitter to build a persistent, queryable map of your repository structure. By pruning irrelevant noise and focusing on actual blast-radius changes, it dramatically reduces token consumption during AI-assisted development.

Read →Share →
[ Analysis + ]

tirth8205/code-review-graph ↗

Technical Analysis

`code-review-graph` is a local-first developer tool designed to optimize AI-assisted coding by creating a structural, persistent map of a repository. By leveraging Tree-sitter for AST parsing, the tool tracks functions, classes, imports, and call sites to build a queryable graph of the codebase. When a file changes, the system performs "blast-radius" analysis to identify only the relevant callers, dependents, and tests affected by the modification. This allows AI assistants to ingest precise, minimal context rather than scanning entire repositories, resulting in significant token savings. Supporting a wide array of programming languages and offering incremental updates in under two seconds, the tool integrates seamlessly with popular AI coding platforms via MCP (Model Context Protocol).

This project is highly beneficial for developers and engineering teams managing large-scale codebases or monorepos, where token costs and context window limitations often hinder AI productivity. By drastically reducing the noise in AI reviews and debugging sessions—achieving a median token reduction of approximately 82x—it transforms how agents interact with complex projects. The tool is trending because it addresses the growing demand for efficient, "local-first" AI infrastructure that protects source code privacy while enhancing accuracy in CI/CD pipelines. Its ability to provide risk-scored PR reviews locally further appeals to teams looking to integrate AI into their merge gates without external data exposure. Ultimately, it provides a performant, automated solution to the "context bloat" problem inherent in modern AI development workflows.

ayghri/i-have-adhd ↗

Technical Analysis

The `i-have-adhd` repository provides a specialized skill plugin for coding agents, such as Claude Code and Codex, designed to optimize LLM responses for efficiency and clarity. By enforcing a strict set of ten rules—such as prioritizing immediate actions, utilizing numbered lists, and eliminating conversational filler like preambles or closers—it forces the AI to cut straight to the technical solution. Technically, the tool functions as a system-prompt modifier that can be installed directly through plugin marketplaces, requiring no local cloning to operate. Users can trigger this behavior explicitly via a command or allow the agent to invoke it implicitly when a task demands high-density information. Additionally, the plugin is highly customizable, allowing users to fork the repository and tweak the core operational rules in `SKILL.md` to suit their specific workflow requirements.

This project is primarily intended for developers who struggle with the verbose, conversational nature of standard AI assistants and prefer a streamlined, "just the facts" interaction model. It is gaining traction because it addresses a common pain point: the frustration of scrolling past excessive polite fluff to find the actual code or command required to solve a problem. By shifting the AI’s communication style to be task-oriented and objective, the plugin minimizes cognitive load and keeps developers in a state of flow. Its popularity highlights a broader movement toward "ADHD-friendly" computing, where users are actively seeking ways to reduce digital distractions and improve the precision of their productivity tools. Ultimately, it serves as an essential utility for anyone looking to maximize their development speed by stripping away unnecessary text.
